## Нефункциональные требования

| Нефункциональное требование | Описание                                                                                                                      | Критичность                                                                                                                       |
|-----------------------------|-------------------------------------------------------------------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------|
| **Производительность**      | Время ответа API < 300 мс для 95% запросов; поддержка до 100K активных пользователей в пиковую нагрузку.                      | **Высокая** — способность проводить массовые события (челленджи, соревнования).                                                   |
| **Доступность**             | SLA ≥ 99.9% для критических сервисов (Auth, Activity Tracking).                                                               | **Высокая** — недоступность блокирует основной функционал: вход и запись тренировок.                                              |
| **Безопасность**            | Шифрование данных в покое и при передаче, поддержка двухфакторной аутентификации, проведение регулярных аудитов безопасности. | **Высокая** — обязательное условие для защиты персональных и health-данных; нарушение ведёт к юридическим и репутационным рискам. |
| **Совместимость**           | Поддержка iOS 15+, Android 10+, протоколов BLE 4.0+ и ANT+.                                                                   | **Средняя** — обеспечивает охват целевой аудитории и работу с популярными устройствами, но не блокирует запуск.                   |
| **Локализация**             | Поддержка 20+ языков, включая корректное отображение форматов даты/времени, валют и единиц измерения.                         | **Средняя** — необходима для глобального масштабирования, но может быть расширена постепенно.                                     |
| **Соблюдение законов**      | Соответствие GDPR, CCPA, LGPD и другим применимым региональным нормам защиты данных.                                          | **Высокая** — юридическое требование для запуска в ключевых регионах; отсутствие соответствия = невозможность выхода на рынок.    |
| **Восстановление**          | Время восстановления (RTO) ≤ 15 мин, максимальная потеря данных (RPO) ≤ 5 мин для критических компонентов.                    | **Высокая** — потеря тренировок или профиля неприемлема для пользователей и подрывает доверие к платформе.                        |
