# Описание сценариев использования приложения

---

### 1. Первый запуск и персонализированный онбординг

**Актор:** Новый пользователь (любитель, новичок)

**Цель:** Быстро начать тренироваться с минимальными усилиями и почувствовать поддержку бренда.

**Основной поток:**

1. Пользователь устанавливает приложение и выбирает способ входа (email, Apple, Google, аккаунт компании).
2. Проходит короткий онбординг (3 шага):
   - Указывает цель: «Поддерживать форму», «Подготовиться к 5K», «Йога для расслабления».
   - Выбирает виды активности (бег, ходьба, йога).
   - Добавляет текущую обувь (опционально — можно пропустить).
3. Система генерирует первую тренировку (например, «Прогулка 20 мин») и предлагает присоединиться к локальному челленджу «Неделя движения».
4. Пользователь запускает тренировку — приложение использует GPS и датчики телефона.

**Альтернативные потоки:**

1. Нет интернета → онбординг доступен в оффлайн-режиме, синхронизация произойдёт позже.

**Связь с бизнес-целями:**

- Повышение вовлечённости с первого контакта.
- Формирование образа бренда как поддерживающего и доступного.

---

### 2. Запись и анализ тренировки

**Актор:** Активный пользователь (любитель, полупрофессионал)

**Цель:** Отслеживать прогресс, получать обратную связь и мотивацию.

**Основной поток:**

1. Пользователь запускает тренировку (бег по маршруту или свободный режим).
2. Приложение записывает: GPS-маршрут, темп, дистанцию, ЧСС (с телефона или wearable).
3. После завершения отображается сводка:
    - График темпа, карта маршрута.
    - Сравнение с последней аналогичной тренировкой («Ты стал быстрее на 5%»).
    - Сравнение с медианой по региону («Ты в топ-30% бегунов в Москве»).
4. Система начисляет очки, открывает бейдж («7 дней подряд!»).
5. Появляется рекомендация: «Твоя обувь пробежала 580 км. Рассмотрите модель X для лучшей амортизации».

**Альтернативные потоки:**

1. Потеря сигнала GPS → приложение интерполирует маршрут и сохраняет данные локально.
2. Пользователь отключает сравнение с регионом → данные не отправляются в аналитику.

**Связь с бизнес-целями:**

- Стимулирование самосоревнования («соревнуйся с собой»).
- Прямая связь активности с рекомендациями товаров → рост продаж.

---

### 3. Поиск партнёра для совместной тренировки

**Актор:** Социально ориентированный пользователь

**Цель:** Найти единомышленников для мотивации и поддержки.

**Основной поток:**

1. Пользователь открывает раздел «Найти напарника».
2. Указывает:
    - Тип активности (бег, йога, велосипед).
    - Радиус поиска (до 5 км).
    - Онлайн-статус («тренируется сейчас» или «готов сегодня»).
3. Система показывает подходящих пользователей с краткой информацией (уровень, интересы, последние активности).
4. Пользователь отправляет приглашение → получает подтверждение.
5. Создаётся временная сессия: общая цель, возможность чата, общая запись тренировки.
6. После завершения оба получают бонус: «Командный дух» (+50 очков).

**Альтернативные потоки:**

1. Нет активных пользователей рядом → система предлагает создать «открытый вызов» для региона.
2. Пользователь скрыл геолокацию → не участвует в поиске.

**Связь с бизнес-целями:**

- Формирование самоподдерживающихся сообществ.
- Укрепление эмоциональной связи с брендом через социальный опыт.

---

### 4. Участие в региональном челлендже

**Актор:** Конкурентоспособный пользователь

**Цель:** Проявить себя, сравниться с другими, получить признание.

**Основной поток:**

1. Пользователь получает уведомление: «Челлендж "Зимний километраж" стартует завтра!».
2. Нажимает «Участвовать» — регистрация автоматическая.
3. В течение 30 дней все пробежки учитываются в общем прогрессе.
4. В личном кабинете видит:
    - Свой суммарный километраж.
    - Место в рейтинге региона.
    - Прогресс команды (если вступил в группу).
5. За промежуточные цели (100 км, 300 км) получает цифровые награды и купоны.
6. По завершении — итоговый сертификат и push: «Ты в топ-10%! Получи скидку 20%».

**Альтернативные потоки:**

1. Пользователь пропустил день → система не исключает, но снижает шансы на лидерство.
2. Челлендж отменён из-за низкой активности → все участники получают утешительный бонус.

**Связь с бизнес-целями:**

- Массовое вовлечение → сетевой эффект.
- Прямая конверсия через таргетированные промоакции.

---

### 5. Обновление спортивного инвентаря

**Актор:** Лояльный клиент бренда

**Цель:** Получить актуальную рекомендацию на новую обувь или экипировку.

**Основной поток:**

1. Система отслеживает пробег по модели обуви (например, «Model X — 620 км»).
2. При 70% износа (≈700 км) отправляет уведомление:
   «Твоя обувь приближается к концу срока. Выбери новую для защиты суставов».
3. Показывает подборку моделей, подходящих под:
    - Тип активности (бег по асфальту).
    - Стиль (амортизация, лёгкость).
    - Бюджет (если указан).
4. Пользователь переходит в e-commerce без повторного логина.
5. Оформляет заказ — данные синхронизируются обратно в профиль.

**Альтернативные потоки:**

1. Пользователь указал, что «ремонтирует обувь» → уведомления откладываются.
2. Нет данных о пробеге → система просит вручную указать износ.

**Связь с бизнес-целями:**

- Персонализированные продажи на основе реального использования.
- Укрепление доверия: рекомендации основаны на заботе о здоровье, а не только на прибыли.

---

### 6. Анонимная тренировка с защитой приватности

**Актор:** Пользователь, ценящий конфиденциальность

**Цель:** Тренироваться без публикации данных.

**Основной поток:**

1. Перед тренировкой пользователь включает «Анонимный режим».
2. Тренировка записывается, но:
    - Маршрут не сохраняется.
    - Данные не попадают в рейтинги.
    - Никакие уведомления не отправляются.
3. Метрики (дистанция, ЧСС) сохраняются только в личную статистику.
4. Пользователь может в любой момент отключить режим.

**Альтернативные потоки:**

1. Пользователь случайно оставил геолокацию включённой → система предупреждает перед стартом.

**Связь с бизнес-целями:**

- Соответствие GDPR/CCPA.
- Повышение доверия к платформе как к этичному партнёру.
