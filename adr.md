# Список ADR (Architectural Decision Records)

| №      | Название решения                                        | Суть                                                                               | Обоснование                                                                             |
|--------|---------------------------------------------------------|------------------------------------------------------------------------------------|-----------------------------------------------------------------------------------------|
| ADR-01 | **Event-Driven Microservices как основная архитектура** | Все сервисы взаимодействуют через асинхронные события (Kafka)                      | Поддержка пиков нагрузки, гибкость команд, соответствие реактивной бизнес-логике        |
| ADR-02 | **Multi-cloud развёртывание без vendor lock-in**        | Использование Kubernetes, Terraform, OpenTelemetry; избегание proprietary-сервисов | Стратегическая независимость, оптимизация TCO, соответствие внутренним ИТ-политикам     |
| ADR-03 | **Раздельные хранилища по типу данных**                 | PostgreSQL (профили), Neo4j (соц. связи), ClickHouse (активность)                  | Максимальная производительность и выразительность для каждой доменной модели            |
| ADR-04 | **Централизованный Auth & Privacy Core**                | Единая точка управления идентификацией и согласиями                                | Обеспечение GDPR/CCPA compliance, упрощение аудита                                      |
| ADR-05 | **Мобильное приложение как primary client**             | Веб-версия не планируется на первых этапах                                         | 95% целевой аудитории использует смартфоны; GPS и датчики доступны только на устройстве |
| ADR-06 | **Оффлайн-first подход для активности**                 | Тренировки записываются локально и синхронизируются при подключении                | Поддержка активности в «диких» местах без сети — ключевое требование пользователей      |
| ADR-07 | **Гибридная геймификация: правила + ML**                | Базовые правила («10 тренировок = бейдж») + ML для персонализированных челленджей  | Баланс между предсказуемостью и персонализацией                                         |
