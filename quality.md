## Атрибуты качества

| Атрибут качества                                   | Описание                                                                                                                                 | Метрики                                                                                                                                                                                                                                                                                                          | Критичность                                                                                                 |
|----------------------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|-------------------------------------------------------------------------------------------------------------|
| **Масштабируемость**                               | Способность системы обрабатывать растущее число пользователей, тренировок и социальных взаимодействий без деградации производительности. | • Поддержка до **10 млн активных пользователей**<br>• Обработка **10 млн+ событий активности/сутки**<br>• Горизонтальное масштабирование сервисов без downtime<br>• Время масштабирования под нагрузку ≤ 5 мин                                                                                                   | **Высокая**<br>(Без масштабируемости невозможен глобальный рост и проведение массовых соревнований)         |
| **Конфиденциальность и безопасность**              | Защита персональных данных (геолокация, здоровье, социальные связи) и соблюдение регуляторных требований (GDPR, CCPA и др.).             | • 100% данных в покое и при передаче — **шифрованы (TLS 1.3+, AES-256)**<br>• Все запросы — через **аутентифицированные сессии с согласием**<br>• Возможность полного **удаления данных за ≤ 30 дней**<br>• Количество инцидентов утечки данных = **0**<br>• Прохождение **ежеквартального аудита безопасности** | **Высокая**<br>(Нарушение приватности ведёт к юридическим санкциям, репутационному ущербу и потере доверия) |
| **Наблюдаемость**                                  | Возможность мониторинга, диагностики и быстрого реагирования на инциденты в распределённой системе.                                      | • **99%+ запросов** — с полной трассировкой (trace ID)<br>• Время обнаружения инцидента (MTTD) ≤ **5 мин**<br>• Время восстановления (MTTR) ≤ **15 мин**<br>• Покрытие ключевых сервисов алертами ≥ **95%**<br>• Единая панель мониторинга (логи, метрики, трассировки)                                          | **Высокая**<br>(Без наблюдаемости невозможно обеспечить SLA и быстро устранять сбои в multi-cloud среде)    |
| **Доступность (Availability)**                     | Готовность системы обслуживать запросы пользователей в любое время.                                                                      | • SLA для критических сервисов (Auth, Activity, Profile) ≥ **99.9%**<br>• Время простоя в месяц ≤ **43 мин**<br>• Отказоустойчивость в рамках одного облака (multi-AZ)                                                                                                                                           | **Средняя**<br>(Кратковременные простои допустимы, но критичны в периоды соревнований)                      |
| **Производительность (Latency)**                   | Скорость отклика системы на действия пользователя.                                                                                       | • 95-й перцентиль времени ответа API ≤ **300 мс**<br>• Загрузка тренировки в приложении ≤ **1.5 с**<br>• Время синхронизации с wearable ≤ **10 с**                                                                                                                                                               | **Средняя**<br>(Задержки снижают UX, но не блокируют основной функционал)                                   |
| **Соблюдение нормативных требований (Compliance)** | Соответствие законодательству по обработке данных, доступности, контенту.                                                                | • Поддержка **GDPR, CCPA, LGPD, ADA (доступность)**<br>• Наличие документов: Privacy Policy, DPA, Cookie Consent<br>• Региональная настройка контента и промо                                                                                                                                                    | **Средняя**<br>(Критично для входа на рынки, но реализуется на этапе Foundation)                            |
| **Гибкость развёртывания (Multi-cloud)**           | Возможность размещения компонентов в разных облачных средах без lock-in.                                                                 | • Отсутствие vendor-специфичного кода в бизнес-логике<br>• Использование **Kubernetes, Terraform, OpenTelemetry**<br>• Возможность переноса сервиса между облаками ≤ **1 неделя**                                                                                                                                | **Низкая**<br>(Важна для долгосрочной стратегии, но не влияет на MVP)                                       |

* Три основных атрибута качества : Масштабируемость, Конфиденциальность и безопасность, Наблюдаемость — все с высокой критичностью.
* Остальные атрибуты важны, но либо реализуются через инструменты (например, производительность — через кэширование), либо не блокируют запуск (например, multi-cloud).
* Метрики привязаны к измеримым целям, что позволяет использовать их в SLI/SLO/SLA и системах мониторинга.

## Нефункциональные требования

| Нефункциональное требование | Описание                                                                                                                      | Критичность                                                                                                                       |
|-----------------------------|-------------------------------------------------------------------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------|
| **Производительность**      | Время ответа API < 300 мс для 95% запросов; поддержка до 100K активных пользователей в пиковую нагрузку.                      | **Высокая** — способность проводить массовые события (челленджи, соревнования).                                                   |
| **Доступность**             | SLA ≥ 99.9% для критических сервисов (Auth, Activity Tracking).                                                               | **Высокая** — недоступность блокирует основной функционал: вход и запись тренировок.                                              |
| **Безопасность**            | Шифрование данных в покое и при передаче, поддержка двухфакторной аутентификации, проведение регулярных аудитов безопасности. | **Высокая** — обязательное условие для защиты персональных и health-данных; нарушение ведёт к юридическим и репутационным рискам. |
| **Совместимость**           | Поддержка iOS 15+, Android 10+, протоколов BLE 4.0+ и ANT+.                                                                   | **Средняя** — обеспечивает охват целевой аудитории и работу с популярными устройствами, но не блокирует запуск.                   |
| **Локализация**             | Поддержка 20+ языков, включая корректное отображение форматов даты/времени, валют и единиц измерения.                         | **Средняя** — необходима для глобального масштабирования, но может быть расширена постепенно.                                     |
| **Соблюдение законов**      | Соответствие GDPR, CCPA, LGPD и другим применимым региональным нормам защиты данных.                                          | **Высокая** — юридическое требование для запуска в ключевых регионах; отсутствие соответствия = невозможность выхода на рынок.    |
| **Восстановление**          | Время восстановления (RTO) ≤ 15 мин, максимальная потеря данных (RPO) ≤ 5 мин для критических компонентов.                    | **Высокая** — потеря тренировок или профиля неприемлема для пользователей и подрывает доверие к платформе.                        |
