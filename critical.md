# Критические бизнес-сценарии

1. **Первый запуск приложения** – пользователь регистрируется, указывает цели, инвентарь, получает первую тренировку.
### 1. Первый запуск и онбординг нового пользователя

**Цель:** Обеспечить быстрое, мотивирующее и персонализированное первое впечатление, чтобы минимизировать отток на раннем этапе.

1. Пользователь устанавливает приложение и проходит регистрацию (через email, соцсеть или аккаунт компании).
2. Проходит короткий онбординг: указывает цель («похудеть», «подготовиться к марафону», «просто двигаться»), уровень подготовки, виды активности.
3. Добавляет текущий спортивный инвентарь (например, модель беговой обуви).
4. Получает персонализированную первую тренировку (например, «лёгкая пробежка 2 км»).
5. Система предлагает присоединиться к локальной группе или челленджу.

### 2. Завершение тренировки и пост-тренировочное взаимодействие

**Цель:** Усилить чувство достижения, предоставить ценную аналитику и стимулировать следующее действие (повтор, покупка, публикация).

1. Пользователь завершает тренировку (бег, йога и т.д.).
2. Приложение показывает сводку: дистанция, темп, ЧСС, сравнение с предыдущими сессиями.
3. Система начисляет очки, открывает бейдж («5 тренировок подряд!»).
4. Появляется персонализированная рекомендация: «Ты улучшил свой темп на 8% — поздравляем! Поделись с друзьями».
5. Данные синхронизируются с облаком и (опционально) с wearables.

### 3. Поиск партнёра для совместной тренировки

**Цель:** Стимулировать социальное взаимодействие и повышение частоты тренировок через совместную активность.

1. Пользователь открывает раздел «Найти напарника».
2. Фильтрует по:
   * Местоположению (радиус 5 км).
   * Виду активности (бег, велосипед).
   * Уровню (новичок, продвинутый).
   * Онлайн-статусу («тренируется сейчас»).
3. Система показывает подходящих пользователей.
4. Пользователь отправляет приглашение → получает подтверждение → создаётся временная «сессия».
5. После тренировки оба участника получают бонус («Командный дух» — +50 очков).

### 4. Участие в региональном челлендже

**Цель:** Создать массовое событие, повышающее вовлечённость и формирующее чувство принадлежности к сообществу бренда.

1. Пользователь видит уведомление: «Челлендж “Зимний километраж” стартует завтра!»
2. Регистрируется — система автоматически отслеживает все пробежки в течение 30 дней.
3. В личном кабинете видит прогресс, рейтинг в регионе, сравнение с друзьями.
4. За промежуточные цели получает цифровые награды и купоны.
5. о завершении — итоговый сертификат, рейтинг, push-уведомление: «Ты в топ-10% — получи скидку 20% на новую куртку».

### 5. Обновление спортивного инвентаря и рекомендация товаров

**Цель:** Превратить данные об активности в коммерческую возможность с высоким уровнем доверия.

1. Система отслеживает пробег по модели обуви (например, «Nike Pegasus 40 — 620 км»).
2. При приближении к порогу износа (700 км) отправляет уведомление: «Твоя обувь подходит к концу пробега. Выбери новую пару для комфорта и безопасности».
3. Показывает подборку моделей, совместимых с типом тренировок пользователя (бег по асфальту → амортизация).
4. Пользователь переходит в e-commerce, добавляет в корзину, оформляет заказ — без повторного логина.
